## 1. 診断結果（すべて 2026-07-07 実測）

### 検索エンジンにほぼインデックスされていない ← アクセスゼロの直接原因

- `site:toknet.info` の結果は **「DOMAIN ERROR」状態の www ホスト 1 件のみ**。記事は 1 本も出てこない
- `"toknet.info"` / `"AI共創レビュー研究所"` の検索でも本サイトは一切ヒットしない（同名 Facebook ページ「得ネット」が出る = ブランド名でも負けている）
- 技術要因は除外済み: robots.txt は search=yes / Allow all、noindex なし、sitemap 191 URL 正常、canonical apex 統一済み

### なぜインデックスされないのか（複合要因、影響度順）

| # | 要因 | 証拠 |
|---|------|------|
| 1 | **コンテンツの大量重複** | 190 記事中 **177 記事が同一タイトルテンプレート**（「SNSで話題の「X」徹底レビュー」）。日付を除くと完全同一タイトルが「ウェアラブル」8 本、「スマートグラス」5 本、「AIボイスレコーダー」5 本…。本文もテンプレート 2 種の穴埋めで、商品情報は Amazon 検索リンクのみ。Google の「スケールされた低品質コンテンツ」判定の教科書的パターンで、**クロールされても index する価値がないと判断される** |
| 2 | **www/apex の二重配信 + www の不健全な状態** | www が 301 されず 200 で同一コンテンツを配信（監査 B11）。インデックス上の唯一のエントリが www の DOMAIN ERROR。シグナルが分散し、壊れた方が記録されている |
| 3 | **立ち上げ期の信頼シグナル不足** | ドメイン運用 ~1 ヶ月、被リンク実質ゼロ、ブランド言及ゼロ。この状態で 1 日 2 本の機械投稿はスパムシグナルにしかならない |
| 4 | ソフト 404（全パス 200） | PR #6 で修正済み（マージ待ち） |

### Instagram がゼロの理由

- **action block で 24 日以上 API 投稿自体が失敗**している（`docs/instagram-recovery.md`）
- ブロック以前も: 全投稿が同一画像 + 定型キャプション。フォロワー獲得の仕組み（リール・ハッシュタグ戦略・相互作用）が無く、リーチはフォロワー数に張り付く
- 結論: **現状の Instagram は流入チャネルとして機能しておらず、当面は回復不能**。復旧手順は instagram-recovery.md、ただし復旧しても「毎回同じ画像の定型投稿」を再開すれば再ブロックされる

## 2. 対策（優先度順）

### P0-A: 今日ユーザーがやる 3 つ（合計 15 分、効果最大）

1. **www→apex 301 の GO を出す**（私が `scripts/setup_www_redirect.py` を実行。Cloudflare 本番変更のため明示 GO が必要と判定された）。または自分で: Cloudflare Dashboard → toknet.info → Rules → Redirect Rules → `http.host eq "www.toknet.info"` → 301 → `concat("https://toknet.info", http.request.uri.path)`
2. **GSC で強制的にクロールさせる**: [Search Console](https://search.google.com/search-console) → toknet.info プロパティ → (a) サイトマップに `https://toknet.info/sitemap-index.xml` を送信、(b) URL 検査でトップ・カテゴリ 9 ページ・手書きレビュー 12 本（macbook-pro-m4-review 等）を「インデックス登録をリクエスト」。**量産記事はリクエストしない**
3. **Bing Webmaster Tools に登録**（未登録なら）: GSC からのインポートで 2 分。日本では Bing 経由の流入も馬鹿にならない

### P0-B: コンテンツモデルの転換（最重要・要 GO）

現行の「12h ごとにテンプレ記事を自動量産」は**続けるほどインデックスから遠ざかる**。方針転換を提案:

1. **量産の停止 or 大幅減速**: cron の記事生成を 1 日 2 本 → 週 2〜3 本に。浮いた枠は既存記事の改善に
2. **重複記事の統合**: 同一タグの記事（ウェアラブル 8 本等）を各 1 本に統合し、残りは draft: true + 301（実装可能。クラスタごとに最良 1 本を残す）
3. **「index される記事」の要件をパイプラインに実装**:
   - タイトルの一意化（同一タグ再訪時は角度を変える: 「選び方」「比較」「Q&A」）
   - 検索需要のあるロングテールへ照準（トレンドタグ単体でなく「<商品名> レビュー」「<商品名> vs <競合>」）
   - 本文に実データ（価格帯・スペック表・比較表）を注入。Amazon 検索リンクの羅列をやめ PA-API または手動キュレーションで実商品を 2〜3 点
4. **手書き 12 本（rating 付き）が資産**: これらは実コンテンツなので、内部リンクのハブ（カテゴリページ→手書きレビュー→量産記事）を張り、GSC で個別リクエスト

### P1: 流入チャネルの再配分

- **X（Twitter）を主 SNS に昇格**: `scripts/post_tweet.py` が既にある。IG と違い API ブロックも無く、テック系の拡散はほぼ X。記事投稿 + トレンドタグ引用を自動化（要 GO で実装）
- **Mastodon は継続**（トークン設定のみで動く。MSTODON_ACCESS_TOKEN が未設定のまま — 設定すれば即流入源）
- Instagram は復旧手順（7〜14 日停止→手動投稿→低頻度再開）を踏み、**リール中心・記事ごとの固有画像**ができるまで自動投稿を再開しない

### P2

- 記事別 OGP 画像生成（SNS CTR + IG 再開の前提）
- はてなブックマーク等への自然な露出（技術系まとめ記事の投稿）
- tag ページ（/tags/<tag>/）で内部リンク網を形成

## 3. 効果の見通し（正直に）

- P0-A だけで「インデックスされ始める」までは行ける（1〜2 週間で site: に記事が出るのが成功指標）
- ただし**順位が付いて流入が発生するかはコンテンツ品質次第**（P0-B）。テンプレ量産のままなら、インデックスされても検索 1 ページ目には出ない
- Instagram は「直す」より「主戦場を X に移す」方が費用対効果が一桁良い
