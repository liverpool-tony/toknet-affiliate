## 6. cronジョブ設計思想

```
[月次] 方向性・カテゴリ・競合 → 戦略ドキュメント更新
  ↓
[週次] GA4トレンド → キュー再生成 → nightlyプロンプト更新
  ↓
[毎時] キューから1タスク → 記事修正 → デプロイ → 報告
  ↓
[朝] 総括レポート → 効果測定 → 翌夜の準備
```

### 自己更新ルール
- 週次レビューでKPI未達の場合、タスク優先度を変更する
- 特定カテゴリのトラフィックが伸びたら、そのカテゴリのタスクを増やす
- 効果の薄いタスクタイプ（例: タイトル変更だけで流入増なし）は縮小する
- 月次レビューでフェーズ移行を判定する
